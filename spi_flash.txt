一、外置 SPI Flash (W25Q64CV, 8MB)
================================================================================

  芯片型号：Winbond W25Q64CV
  容量：    8MB (8,388,608 bytes)
  接口：    SPI2 (由 SFUD 库驱动)
  文件：    sfud/inc/sfud_cfg.h (第45行)
            配置为 {.name = "W25Q64CV", .spi.name = "SPI1"} (注意：SPI名称配置为SPI1但实际使用SPI2外设)
            CS引脚：PE4 (SPI_FLASH_CS)

  SPI Flash 上的存储分为两个区域：

  ┌──────────────────────────────────────────────────────────────────┐
  │  SPI Flash (W25Q64CV, 8MB) 存储布局                               │
  ├──────────────────────────────────────────────────────────────────┤
  │  地址 0x000000 ~ 0x1FFFFF (0 ~ 2MB)                              │
  │  · EasyFlash ENV 环境变量区                                       │
  │    · 起始地址：EF_START_ADDR = 0 (ef_cfg.h:95)                    │
  │    · 区域大小：ENV_AREA_SIZE = 2MB (ef_cfg.h:97)                   │
  │    · 擦除粒度：EF_ERASE_MIN_SIZE = 4KB (ef_cfg.h:59)              │
  │    · 写粒度：  EF_WRITE_GRAN = 32bit (ef_cfg.h:66)               │
  │    · 模式：    NG (Next Generation) 模式 (EasyFlash V4.x)         │
  ├──────────────────────────────────────────────────────────────────┤
  │  地址 0x200000 ~ 0x6FFFFF (2MB ~ 7MB)                            │
  │  · 保留/未使用区域                                                │
  ├──────────────────────────────────────────────────────────────────┤
  │  地址 0x700000 ~ (7MB 起)                                        │
  │  · ARB 任意波形数据区                                             │
  │    · 直接通过 SFUD API 读写                                       │
  │    · 每个 ARB 波形占用 16KB (1024×16 bytes)                       │
  │    · 索引 1 起始地址：7MB + 0×16KB = 0x700000                    │
  │    · 索引 2 起始地址：7MB + 1×16KB = 0x704000                    │
  │    · ...                                                         │
  │    · 文件：APP/save_load.c (第76-79行, 第89-98行)                  │
  └──────────────────────────────────────────────────────────────────┘

  1.1 EasyFlash ENV (环境变量) 区域 (0 ~ 2MB)
  ─────────────────────────────────────────────

  EasyFlash 在 SPI Flash 上实现了一个键值对 (Key-Value) 数据库，
  类似于 EEPROM 的功能。使用 NG (Next Generation) 模式，支持磨损均衡和
  垃圾回收 (GC)。

  默认环境变量 (在 ef_port.c:38-44 定义，首次初始化时写入)：

    ┌──────────────────────────┬─────────┬──────────────────────────────┐
    │ 键名 (Key)               │ 默认值  │ 说明                         │
    ├──────────────────────────┼─────────┼──────────────────────────────┤
    │ "iap_need_copy_app"      │ "0"     │ IAP 升级：是否需要拷贝固件   │
    │ "iap_copy_app_size"      │ "0"     │ IAP 升级：拷贝固件的大小      │
    │ "stop_in_bootloader"     │ "0"     │ 是否停留在 Bootloader        │
    │ "device_id"              │ "1"     │ 设备 ID                      │
    │ "boot_times"             │ "0"     │ 系统启动次数计数器            │
    └──────────────────────────┴─────────┴──────────────────────────────┘

  运行时动态环境变量 (由 APP 代码读写)：

    ┌──────────────────────────┬───────────────────────────────────────┐
    │ 键名 (Key)               │ 说明                                  │
    ├──────────────────────────┼───────────────────────────────────────┤
    │ "boot_times"             │ 系统启动次数，每次开机自增 1           │
    │                          │ 文件：APP/save_load.c:24-25           │
    ├──────────────────────────┼───────────────────────────────────────┤
    │ "wave_1" ~ "wave_20"     │ 双通道波形参数 (ChannelParams_t × 2)  │
    │                          │ 共 20 组存储槽位 (save_page 0~4,      │
    │                          │ 每页 4 个位置)                        │
    │                          │ 包含：频率/幅度/偏置/占空比/相位/波形 │
    │                          │ 类型                                  │
    │                          │ 文件：APP/save_load.c:43-63           │
    │                          │       GUI/ui_storage.c:107-111        │
    └──────────────────────────┴───────────────────────────────────────┘

  每个 "wave_N" 键存储的数据结构 (ChannelParams_t × 2 = 两个通道)：

    ChannelParams_t 结构体成员 (定义于 APP/waveform_generator.h)：
      freq     (uint32_t)  频率 (Hz)，默认 1000
      amp      (uint32_t)  幅度 (mV)，默认 1650
      offset   (uint32_t)  偏置 (mV)，默认 1650
      duty     (uint8_t)   占空比 (%)，默认 50
      phase    (uint8_t)   相位 (°)，默认 0
      type     (uint8_t)   波形类型 (WAVE_SINE, WAVE_RECT, 等30+种)

  注：save_flag 相关的代码 (ef_get_env_blob/ef_set_env_blob 操作
      SAVE_FLAG_KEY) 已被注释掉，当前未使用。
      文件：APP/save_load.c:5,33,81-83

  1.2 ARB 任意波形数据区 (7MB 起)
  ─────────────────────────────────

  此区域绕过 EasyFlash，直接通过 SFUD API 读写原始二进制数据。

    存储路径：
      PC端 → USB CDC → host_device.c (Recv_data_dac[16000])
           → divide_dac_data_by_4() (数据右移2位/除以4)
           → save_arb(index) → sfud_erase_write()

      MCU端 ← load_arb(index, ch) ← sfud_read()
           ← gui_menu.c → Waveform[ch][8190]

    地址计算：
      addr = 7 × 1024 × 1024 + (index - 1) × 1024 × 16
      每个 ARB 波形数据大小 = USB_BUFFER_SIZE (32000 bytes = 32KB)

    调用位置：
      · 保存：host_device.c:97 → save_arb(ARB_num)
        (USB 收到 "DDS_WAVE" 命令后触发)
      · 加载：GUI/ui_menu.c:571 → load_arb(n, selected_ch)
        (菜单选择加载 ARB 波形时触发)

  IAP 固件升级 (EasyFlash IAP)
  ─────────────────────────────────

  开启 EF_USING_IAP (ef_cfg.h:53)，允许在 SPI Flash 上暂存固件升级包，
  实现 Bootloader 升级功能。相关环境变量：
    "iap_need_copy_app"  — 是否需要从备份区拷贝固件到用户区
    "iap_copy_app_size"  — 固件大小
    "stop_in_bootloader" — 是否停留在 Bootloader

  相关 API (easyflash.h:66-80)：
    ef_erase_bak_app()       — 擦除备份区
    ef_write_data_to_bak()   — 写入固件到备份区
    ef_copy_app_from_bak()   — 从备份区拷贝到用户区
    ef_erase_user_app()      — 擦除用户 APP 区
